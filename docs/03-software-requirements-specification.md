## 2. 引言

### 2.1 编写目的

本文档用于明确国有企业综合管理系统平台 V1 的功能需求、非功能需求、外部接口、约束条件和验收依据，作为需求、设计、开发、测试和上线的统一基线。

### 2.2 项目背景

当前企业内部存在流程分散、数据不统一、审批效率低、审计追溯弱等问题。项目目标是建设统一平台底座，并逐步扩展人事、OA、资产、合同、采购等模块，形成集中部署、统一权限、统一留痕的企业管理平台。

### 2.3 术语定义

| 术语 | 定义 |
| --- | --- |
| 平台底座 | 提供认证、权限、组织、日志、文件、消息、流程等通用能力 |
| 工作流 | 审批流程的定义和运行机制 |
| 数据权限 | 用户可查看或操作的数据范围 |
| 待办 | 当前用户需要处理的任务或通知 |

## 3. 总体描述

### 3.1 产品定位

本系统是面向国有企业的综合管理平台，提供统一入口、统一权限、统一流程、统一审计、统一附件和统一消息能力。

### 3.2 产品目标

- 建立统一平台底座
- 支撑千级用户规模
- 满足国企权限和审计要求
- 具备模块化可扩展能力
- 支持 Windows 客户端形态

### 3.3 用户特征

- 普通员工：基础业务办理和待办处理
- 管理人员：审批处理、统计查看
- 管理员：系统维护、参数配置、权限分配
- 审计人员：日志、留痕和数据变更检查

### 3.4 运行环境

- 客户端：Windows 10 或 Windows 11
- 服务端：Linux
- 数据库：PostgreSQL 或 MySQL 8
- 浏览器内核：Electron 内嵌 Chromium

## 5. 非功能需求

### 5.1 性能需求

- 普通查询接口平均响应时间不超过 3 秒
- 复杂报表接口平均响应时间不超过 10 秒
- 支持千级用户规模
- 并发在线用户按 300 至 1000 进行容量规划

### 5.2 可用性需求

- 系统年度可用性目标不低于 99.5%
- 支持应用异常恢复和服务重启
- 支持数据库和文件定时备份

### 5.3 安全性需求

- 密码加密存储
- 登录失败锁定
- 菜单、按钮、数据权限控制
- 审计日志全链路留痕
- 文件上传白名单
- 防 SQL 注入、XSS、CSRF

### 5.4 可维护性需求

- 模块化设计
- 接口规范统一
- 日志规范统一
- 错误码统一
- 支持单元测试和集成测试

### 5.5 可扩展性需求

- 平台可扩展至合同、采购、预算、供应商等模块
- 新模块统一复用权限、流程、消息、附件能力

### 5.6 兼容性需求

- 支持 Windows 10 和 Windows 11
- 数据库可适配 PostgreSQL 和 MySQL
- 支持内网部署环境

## 6. 约束条件

- 客户端交付形态需为 Windows 安装软件
- 服务端采用集中部署方式
- 优先采用成熟稳定开源技术栈
- 为单点登录、国产数据库、信创适配预留空间

## 7. 外部接口需求

- LDAP 或 AD 单点登录预留
- 邮件和短信网关预留
- 企业微信或钉钉通知预留
- 文件存储支持 MinIO 或 NAS
- 工作流引擎采用 Flowable

## 8. 验收标准

- 平台底座功能完整可用
- 三个业务模块具备端到端流程能力
- 权限、流程、文件、消息、日志联动正常
- 可生成 Windows 安装包并完成内网部署
