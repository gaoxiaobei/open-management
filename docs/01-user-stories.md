## 2. 文档目的

本文档用于从业务角色视角描述系统目标、使用场景和验收结果，为功能规格说明书和软件需求规格说明书提供输入依据。

## 3. 角色定义

| 角色 | 说明 |
| --- | --- |
| 系统管理员 | 负责系统配置、组织架构、权限、参数、日志管理 |
| 人力资源专员 | 负责员工档案、入转调离、岗位信息维护 |
| 普通员工 | 登录系统、发起申请、查看个人信息和待办 |
| 部门负责人 | 审批本部门流程，查看本部门数据 |
| 分管领导 | 审批关键事项，查看分管范围统计 |
| 资产管理员 | 维护资产台账，处理领用、归还、维修、报废 |
| 审计人员 | 查看日志、审批痕迹、数据变更记录 |
| 运维人员 | 监控服务状态、异常日志、系统参数 |

## 4. 优先级定义

| 优先级 | 说明 |
| --- | --- |
| P0 | 第一阶段必须实现，否则平台不可交付 |
| P1 | 第一阶段建议实现，用于形成完整示范闭环 |
| P2 | 后续迭代实现，不纳入第一阶段验收 |

## 5. 平台底座用户故事

### 5.1 US-001 登录认证

| 项目 | 内容 |
| --- | --- |
| 编号 | US-001 |
| 优先级 | P0 |
| 角色 | 普通员工 |
| 用户故事 | 作为普通员工，我希望通过工号和密码登录系统，以便进入系统处理业务和查看待办。 |

验收标准：

- 支持账号密码登录
- 支持验证码
- 登录成功进入首页
- 登录失败提示原因
- 连续失败达到阈值后锁定账号

### 5.2 US-002 动态菜单权限

| 项目 | 内容 |
| --- | --- |
| 编号 | US-002 |
| 优先级 | P0 |
| 角色 | 普通员工、部门负责人、管理员 |
| 用户故事 | 作为不同岗位的用户，我希望登录后只看到自己有权限的菜单，以便避免误操作和越权访问。 |

验收标准：

- 菜单基于角色动态加载
- 无权限菜单不可见
- 直接访问无权限页面时返回无权访问提示
- 按钮权限与菜单权限联动

### 5.3 US-003 数据权限隔离

| 项目 | 内容 |
| --- | --- |
| 编号 | US-003 |
| 优先级 | P0 |
| 角色 | 部门负责人、分管领导、审计人员 |
| 用户故事 | 作为管理角色，我希望只查看授权范围内的数据，以便保证数据隔离与合规访问。 |

验收标准：

- 支持仅本人、本部门、本部门及下级、全部等数据权限范围
- 列表查询自动带入权限条件

## 6. 业务模块用户故事

### 6.1 US-101 员工档案维护

| 项目 | 内容 |
| --- | --- |
| 编号 | US-101 |
| 优先级 | P1 |
| 角色 | 人力资源专员 |
| 用户故事 | 作为人力资源专员，我希望录入和维护员工档案，以便保持员工信息准确完整。 |

验收标准：

- 支持新增、编辑、查询员工档案
- 支持身份证号、手机号等格式校验
- 支持在职、离职、试用等状态管理
- 支持档案附件上传

### 6.2 US-102 请假申请

| 项目 | 内容 |
| --- | --- |
| 编号 | US-102 |
| 优先级 | P1 |
| 角色 | 普通员工 |
| 用户故事 | 作为普通员工，我希望在线发起请假申请，以便由系统自动流转审批。 |

验收标准：

- 支持选择请假类型、时间和事由
- 自动校验起止时间
- 支持上传证明材料
- 提交后生成审批流程
- 支持撤回未审批流程

### 6.3 US-103 资产领用

| 项目 | 内容 |
| --- | --- |
| 编号 | US-103 |
| 优先级 | P1 |
| 角色 | 普通员工、资产管理员 |
| 用户故事 | 作为普通员工，我希望在线申请领用办公资产，以便规范资产发放流程。 |

验收标准：

- 支持从资产目录选择可用资产
- 提交后进入审批
- 审批通过后资产状态更新为已领用
- 记录领用人、时间和归还状态
