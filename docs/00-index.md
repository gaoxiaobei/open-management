## 4. 文档维护规则

- 需求变更先更新用户故事和功能规格，再同步 SRS
- 设计变更必须同步更新架构、数据库、模块、接口四类设计文档
- 任何变更进入开发基线前，必须补充修订记录并明确版本号升级
- 同一业务能力的术语、状态定义、编码规则在所有文档中保持一致

## 5. 当前基线说明

当前文档包为项目启动阶段的 `V1.0` 评审基线，用于：

- 立项汇报
- 需求澄清
- 技术方案评审
- 后续脚手架和详细设计展开
